# RF Local / B1 Map Processing Pipeline

This folder contains a Bash pipeline for generating an **RF local / B1 map** from DREAM acquisition data.

The script processes the STE and FID volumes, extracts the nominal flip angle from the Bruker `method` file, concatenates the DREAM volumes, and runs `qi dream` to generate the B1 map.


The resulting B1 map can be used as the **RF local map** required for B1 correction in the MTSat pipeline.

## Required input files

| File | Description |
|---|---|
| `STE.nii.gz` | Stimulated echo volume from the DREAM acquisition |
| `FID.nii.gz` | Free induction decay volume from the DREAM acquisition |
| `method` | Bruker method file from the B1/DREAM acquisition |

## Requirements

The following tools must be installed and available in your system `PATH`:

- MRtrix3
  - `mrcat`
- QUIT / QI
  - `qi`

## Usage

```bash
./RFlocal.sh -s STE.nii.gz -f FID.nii.gz -m method [options]
```

## Required arguments

| Argument | Description |
|---|---|
| `-s` | STE volume, for example `STE.nii.gz` |
| `-f` | FID volume, for example `FID.nii.gz` |
| `-m` | Bruker `method` file from the B1/DREAM acquisition |

## Optional arguments

| Argument | Description | Default |
|---|---|---|
| `-o` | Output prefix | `prefixdream` |
| `-h` | Show help message | — |

## Example

```bash
./RFlocal.sh \
  -s STE.nii.gz \
  -f FID.nii.gz \
  -m method \
  -o prefixdream
```

## Pipeline details

### 1. Extract `SteamPulse`

The script searches for the `SteamPulse` value inside the Bruker `method` file:

```text
##$SteamPulse=
```

This value is used as the nominal flip angle input for the DREAM B1 calculation.

### 2. Concatenate FID and STE volumes

The FID and STE volumes are concatenated into a single 4D file using `mrcat`:

```text
dream_file.nii.gz
```

The script concatenates the volumes in the following order:

```text
FID + STE
```

### 3. Generate B1 map

The B1 map is generated using:

```bash
qi dream dream_file.nii.gz --out=prefixdream --order=f -a <SteamPulse>
```

where `<SteamPulse>` is the value extracted from the Bruker `method` file.

## Intermediate files

The pipeline generates the following intermediate file:

```text
dream_file.nii.gz
```

## Output

The main output is the B1 map generated by `qi dream`.

Using the default output prefix, the expected outputs are:

**prefixdream_B1map.nii.gz**

**prefixdream_angle.nii.gz**

where:

prefixdream_B1map.nii.gz contains the relative B1 / RF local map.

prefixdream_angle.nii.gz contains the estimated flip angle map.



## Important notes

- The STE and FID volumes must come from the same DREAM acquisition.
- The Bruker `method` file must correspond to the same DREAM acquisition.
- The output B1 map is used as the RF local map for the MTSat pipeline.
- Visual inspection of the generated B1 map is recommended before using it for MTSat correction.
- Running the script multiple times in the same folder may overwrite `dream_file.nii.gz`.
